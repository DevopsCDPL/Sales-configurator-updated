CRITICAL TOOLING LESSON: full-project `tsc -p` TIMES OUT (>45s) in sandbox; piping tsc through head/tail MASKS exit codes → false passes → production build failures. CORRECT: `timeout 43 npx tsc --noEmit --jsx react-jsx --strict --esModuleInterop --skipLibCheck --moduleResolution node --module esnext --target ES6 --lib dom,dom.iterable,esnext --resolveJsonModule --allowSyntheticDefaultImports <changed files> > /tmp/o 2>&1; echo EXIT:$?` — targeted files, captured exit. Host Edit/Write TRUNCATE large mounted files + the Linux mount serves STALE reads — use python3 byte-exact patching (CRLF-aware) and verify file tails (`tail -3`) in the SAME bash call. NEW imports must go at the TOP of files (CRA import/first is a build error, tsc won't catch it).
